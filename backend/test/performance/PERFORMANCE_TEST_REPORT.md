## 🔍 性能瓶颈分析

### 已识别的潜在瓶颈

#### 1. 游戏流程中的 API 调用问题 ⚠️
**问题：** 测试中发现 `tileSet.drawTile()` 和 `tilesRemaining()` 方法不存在
**实际 API：** 应使用 `drawOne()` 和 `remaining` 属性
**影响：** 游戏流程测试失败
**建议：** 需要统一 API 或更新测试代码

#### 2. 高频操作下的性能退化
**场景：** 连续 5000+ 次出牌/摸牌操作
**观察：** 操作数量显示为 0，可能由于游戏提前结束
**建议：** 
- 优化游戏循环逻辑
- 增加游戏状态检查
- 确保游戏不会意外结束

#### 3. 内存管理
**观察：** 长时间运行测试中内存增长需要持续监控
**建议：**
- 定期执行 GC
- 监控 Map/Set 数据结构大小
- 及时清理断线玩家数据
